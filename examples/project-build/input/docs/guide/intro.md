# Getting Started Guide

The build command is intended for documentation sets.

1. Put Markdown files under a project root.
2. Configure `inputs`, `rootDir`, and `outDir`.
3. Run `marky build`.

## Why this helps

| Before | After |
| --- | --- |
| Export files one by one | Build the docs tree at once |
| Flattened outputs | Paths preserved under `outDir` |
| Manual browser settings | Shared config controls rendering |
